# Проект рефакторинга чистового кода диплома

## Цель

Сделать `/Users/a.i.semenov/diploma-final` читаемой чистовой поставкой: production-путь отделен от исследовательских сравнений, каждый контракт и каждая функция снабжены русскими поясняющими комментариями, устаревшие файлы удалены, а результаты по gas и корректности не ухудшены.

## Границы изменений

1. Математические формулы, константы кривых, ABI финальных контрактов и Yul hot path не переписываются ради стиля.
2. Допустимы только безопасные изменения: перенос файлов, обновление импортов, русские NatSpec-комментарии, поясняющие комментарии внутри сложных функций, удаление доказанно неиспользуемых файлов и синтаксические упрощения без изменения байткода критического пути.
3. Исследовательские реализации сохраняются, но переносятся из production `src/` в отдельные каталоги `research_variants/`.

## Целевая структура

- `arithmetic/mnt4_3limb/src/`: только используемая MNT4-арифметика.
- `arithmetic/mnt4_3limb/research_variants/`: Barrett, FIOS, Comba, branchless reduction, lazy-reduction варианты и проигравшие low-level эксперименты.
- `arithmetic/lollipop305_2limb/src/`: выбранная 2-limb арифметика и расширения.
- `arithmetic/lollipop305_2limb/research_variants/`: альтернативные 2-limb варианты.
- `arithmetic/mnt6_3limb/src/`: используемая MNT6-арифметика.
- `implementations/*/src/`: только основные проверяющие контракты и необходимые hot-path библиотеки.

## Комментарии

Для каждого файла Solidity добавляются:

1. русское описание назначения библиотеки или контракта;
2. пояснение роли в pipeline;
3. комментарии к константам: поле, модуль, Montgomery-представление, размеры packed-блоков, домены хеширования;
4. русские `@notice` и `@dev` у внешних и ключевых внутренних функций;
5. локальные комментарии перед сложными Yul-блоками и этапами Miller loop/final exponentiation.

## Проверка

После переноса выполняются:

- поиск дубликатов и неиспользуемых файлов;
- `./scripts/run_all.sh`;
- сравнение ключевых gas-строк с предыдущим логом `.reports/run_all_refactor_with_backends.log`;
- удаление временных `out/cache/target/broadcast`.
