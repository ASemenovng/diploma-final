# Очистка модулей MNT4 Merkle/FRI

## 1. Цель

Привести Merkle/FRI-часть проекта к той же структуре, которая уже используется
для остальных модулей диплома:

- в основном каталоге оставить только выбранную исследовательскую линию;
- воспроизводимые, но не выбранные эксперименты перенести в
  `implementations/research_variants/`;
- не удалять историю экспериментов и не смешивать ее с защищаемым итоговым
  результатом;
- сохранить возможность одной командой пересобрать отчеты и повторить тесты.

## 2. Граница результата

Актуальным результатом Merkle/FRI-направления считается не готовый
Solidity-verifier, а математическая спецификация и воспроизводимая Rust-модель
стоимости блочно-сжатой проверки цикла Миллера через Merkle и обычный FRI.

Экспериментальная микротрасса Merkle/DEEP-FRI остается полезным отрицательным
результатом: она демонстрирует стоимость консервативной пошаговой проверки, но
не объявляется оптимальным или production-ready verifier-ом.

## 3. Итоговая структура

```text
implementations/

│   ├── README.md
│   ├── docs/
│   ├── rust/native_fri_cost_model/
│   ├── artifacts/
│   └── scripts/run_cost_model.sh
└── research_variants/
    ├── README.md
    └── mnt4_merkle_deep_fri_microtrace/
        ├── README.md
        ├── docs/
        ├── foundry.toml
        ├── src/
        ├── test/
        ├── rust/microtrace_backend/
        └── scripts/run_report.sh
```

## 4. Перенос актуального модуля

Каталог

```text
implementations/mnt4_merkle_fri_block_compressed/
```

переименовывается в

```text
implementations/mnt4_merkle_fri_cost_model/
```

Новое имя отражает фактическое содержимое: модуль содержит спецификацию,
Rust-калькулятор, отчет и артефакт модели стоимости, но не содержит
production-контракт.

## 5. Перенос исследовательского варианта

Каталог

```text
implementations/mnt4_merkle_deep_fri/
```

переносится в

```text
implementations/research_variants/mnt4_merkle_deep_fri_microtrace/
```

В нем сохраняются:

- основной контракт `MNT4MerkleDeepFriVerifier`;
- три необходимые библиотеки для поля, Merkle-проверки и transcript;
- Foundry-тесты;
- Rust-бэкенд генерации fixtures;
- воспроизводимые tracked-артефакты;
- отчет и независимый аудит;
- скрипт запуска.

Архивный каталог получает явную маркировку: это исследовательский
воспроизводимый прототип, а не рекомендуемый финальный путь.

## 6. Очистка пересобираемых файлов

После переноса удаляются только каталоги, которые исключены из git и
восстанавливаются стандартными командами:

```text
out/
cache/
rust/microtrace_backend/target/
```

Tracked fixtures не удаляются без проверки их использования тестами.

## 7. Документация и комментарии

Нужно обновить:

- корневой `README.md`;
- `WORK_CONTEXT.md`;
- ссылки в актуальных документах и скриптах;
- относительные Foundry remappings и пути скриптов после переноса.

В архивных Solidity-файлах добавляются русские комментарии уровня контракта,
библиотеки и основных этапов проверки. Изменения комментариев не должны менять
ABI, алгоритм или газовые метрики.

## 8. Проверка

После переноса выполняются:

1. `git diff --check`;
2. поиск битых актуальных ссылок на старые каталоги;
3. `cargo test --release` для актуальной Rust-модели стоимости;
4. `scripts/run_cost_model.sh` для актуальной модели;
5. `cargo test --release` для архивного Rust-бэкенда;
6. `forge test -vv` для архивного Foundry-модуля;
7. `scripts/run_report.sh` для архивного воспроизводимого отчета.

Исторические документы в `docs/superpowers/` могут сохранять старые пути,
если они явно относятся к этапу разработки, а не к текущей структуре проекта.

## 9. Критерии готовности

Рефакторинг завершен, если:

- основной модуль называется `mnt4_merkle_fri_cost_model`;
- экспериментальная микротрасса находится только в `research_variants`;
- основной каталог не содержит неподтвержденных production-обещаний;
- архивный модуль запускается из нового пути;
- актуальная модель стоимости пересобирается одной командой;
- Foundry- и Rust-тесты проходят;
- в поддерживаемых README и скриптах нет ссылок на удаленные пути;
- рабочее дерево не содержит пересобираемых `target/out/cache`.
