# Дизайн устранения замечаний полного аудита

Дата: 1 июня 2026 года.

## Цель

Закрыть подтвержденные технические разрывы практической части без изменения
выбранной исследовательской постановки: фиксированные G2-точки и подготовленные
кэши допустимы как доверенная конфигурация, а пользовательские G1-точки должны
проверяться контрактом.

## 1. Граница доверия fixed-G2 режима

Для MNT4 и MNT6 используются две фиксированные G2-точки `Q` и `S`. Их
подготовленные линии хранятся в code-shards и задаются при развертывании.
Контракт не доказывает заново отношение `Q -> lines -> shards`: это сознательная
граница выбранного fixed-G2 эксперимента.

Пользователь не может заменить адреса shards во время вызова. В документации
режим называется доверенной предрегистрацией фиксированного кэша.

## 2. Валидация пользовательских G1-точек

Публичные verifier-функции получают `P` и `R`. Перед тяжелой арифметикой они
проверяют:

1. каноничность limb-представления координат;
2. уравнение короткой формы Вейерштрасса:

   ```text
   y^2 = x^3 + a*x + b.
   ```

Для `G1(MNT4-753)` и `G1(MNT6-753)` кофактор равен `1`, поэтому любая
канонически представленная точка на кривой уже принадлежит требуемой
подгруппе. Отдельное дорогое вычисление `[r]P = O` не нужно.

Проверки добавляются в production-like fixed-shards verifier-ы и измеряются
отдельно, чтобы показать их вклад в gas.

## 3. MNT6 fixed-shards Article640 verifier

Создается симметричный MNT4-контур:

```text
verifyEquationResidueFixedShards(P, R, c, cInv) -> bool
```

Контракт фиксирует в конструкторе:

- координаты второй G2-точки `S`;
- `qXOverTwist`, `qYOverTwist` для фиксированной `Q`;
- `sXOverTwist`, `sYOverTwist` для фиксированной `S`;
- code-shards подготовленных double/add коэффициентов обеих точек.

Вызов:

1. отклоняет некорректные `P` и `R`;
2. проверяет `c * cInv = 1`;
3. вычисляет общий Article640 accumulator для
   `e(P,Q) * e(-R,S) = 1`;
4. возвращает `true` только если итоговый элемент `Fq6` равен единице.

Горячий путь использует packed scratch arena, pointer swap и потоковый
`EXTCODECOPY`. В цикле не создаются временные динамические `bytes`, `Fq3` и
`Fq6` структуры.

## 4. MNT6 Rust fixture

Rust backend строит нетривиальное корректное уравнение:

```text
P = 2 * G1,
R = G1,
Q = G2,
S = 2 * G2.
```

По билинейности:

```text
e(P,Q) = e(R,S).
```

Backend выпускает:

- две пары prepared blobs для `Q` и `S`;
- twist-координаты `Q` и `S`;
- `c`, `cInv` для Article640 residue relation;
- ожидаемый результат уравнения;
- fixture для Solidity differential-тестов.

## 5. MNT-cycle accounting

Исправляются счетчики:

| Кривая | Удвоения | Сложения |
|---|---:|---:|
| MNT4 | 376 | 124 |
| MNT6 | 376 | 123 |

Результаты маркируются как ручная модель multiplication constraints, а не как
скомпилированный circuit.

## 6. Наивная Tate cost model и полный reference baseline

Baseline разделен на два слоя:

1. Наивная cost model: измеренные общие микроблоки и строгая нижняя
   экстраполяция полного Tate-пути.
2. Полностью исполняемый оптимизированный MNT4 reference-контур с arkworks
   cross-check.

Наивный путь намеренно не использует:

- ate-shortening;
- prepared lines;
- sparse multiplication;
- Frobenius decomposition;
- оптимизированные фиксированные цепочки экспоненты;
- Yul hot path.

Полный неоптимизированный Tate-вызов не называется измеренным: число порядка
`2.55B gas` явно маркируется как строгая нижняя экстраполяция.

## 7. Merkle/FRI статус и root runner

Основным результатом остается последняя ordinary-FRI cost model:

```text
implementations/mnt4_merkle_fri_cost_model
```

Архивный DEEP-FRI прототип не возвращается в production path.
`scripts/run_all.sh` запускает ordinary-FRI cost model. Отдельный audit-скрипт
может дополнительно запускать архивный воспроизводимый отрицательный
эксперимент.

## 8. Критерии готовности

1. MNT4 fixed-shards отклоняет точки вне кривой до тяжелой арифметики.
2. MNT6 fixed-shards verifier возвращает `bool` для нетривиального корректного
   fixture и отклоняет подмены `P`, `R`, `cInv`, shards.
3. MNT6 calldata и fixed-shards пути дают одинаковый математический результат.
4. Исправленные MNT-cycle числа подтверждены Rust-тестами.
5. Naive Tate cost model имеет отдельный воспроизводимый запуск, а полный
   исполняемый MNT4 reference-контур отделен от нее явно.
6. Корневой runner запускает ordinary-FRI cost model.
7. Повторный полный аудит проходит без регрессий.

## 9. Уточнения после реализации

Два первоначальных предположения были скорректированы после проверки кодом и
Rust-моделью.

1. Короткое MNT4-style `c`-отношение нельзя механически переносить на MNT6.
   Для MNT6 соответствующий показатель не дает требуемого обратимого
   преобразования в нужной подгруппе. Поэтому production-like MNT6 fixed-shards
   verifier использует полную оптимизированную Frobenius/w0 финальную
   экспоненту. Экспериментальный residue-фрагмент остается только диагностикой.
2. Полный неоптимизированный Tate-вызов отделен от наивной модели стоимости.
   Модель измеряет математически корректные микроблоки и дает строгую нижнюю
   экстраполяцию порядка `2.55B gas`. Полный исполняемый reference-контур
   остается в `implementations/full_onchain_mnt4`.
