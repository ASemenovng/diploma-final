# Привязка кэша линий lollipop-305

## Цель

Добавить к исследовательскому lollipop-305 pipeline два безопасных режима работы с
подготовленными линиями:

1. `calldata + commitment`: пользователь передает blob линий, а контракт перед
   вычислением проверяет его хеш;
2. `fixed code-shards`: адреса data-контрактов фиксируются в конструкторе, а
   проверяющий контракт читает blob инструкцией `EXTCODECOPY`.

## Область применения

Режимы относятся к фиксированным подготовленным наборам линий. Они не заменяют
доказательство корректности кэша для произвольной точки `Q`. Их задача уже:
не позволить пользователю незаметно заменить кэш, который был проверен и
зарегистрирован при развертывании verifier-а.

## Форматы

| Часть | Размер blob |
|---|---:|
| stick | `57,312` байт |
| `E_cycle` | `115,488` байт |
| `Ehat` | `99,840` байт |

## Архитектура

Арифметика и цикл Миллера остаются в `Lollipop305Article640Verifier`. В нем
добавляются внутренние функции для трех residue-проверок, чтобы обертки могли
переиспользовать ровно тот же вычислительный путь без внешнего вызова и без
дублирования формул.

`Lollipop305CommittedCacheVerifier` хранит три неизменяемых commitment-а. Для
каждого blob используется доменно-разделенный хеш:

```text
H(domain || blob.length || keccak256(blob)).
```

`Lollipop305FixedShardsVerifier` хранит списки адресов data-контрактов. Пользователь
не может заменить эти адреса в вызове. Контракт собирает blob из runtime-кода
инструкцией `EXTCODECOPY` и передает его тому же внутреннему проверяющему пути.

## Проверка

Foundry-тесты должны подтвердить:

1. оба режима принимают корректные fixtures Rust-бэкенда;
2. commitment-режим отклоняет подмену каждого из трех blob;
3. fixed-shards режим отклоняет кэш, испорченный до фиксации адресов;
4. оба режима возвращают одинаковый результат;
5. gas-report содержит отдельные значения для трех частей;
6. отчет сравнивает также длину calldata и ее внутреннюю стоимость по правилам
   EVM: `4 gas` за нулевой байт и `16 gas` за ненулевой.

## Итоговый выбор

В `src/` остается fixed code-shards режим как производственный fixed-cache путь,
если он дешевле или сопоставим по полной стоимости вызова. Commitment-режим
сохраняется в `research_variants/` только для воспроизводимого сравнения.
