# Задание: реализовать максимально дешевое 3-limb умножение в Solidity/Yul

Нужно с нуля реализовать максимально экономное по gas умножение элементов
фиксированного простого поля в Solidity/Yul.

Используйте любой алгоритм и любые допустимые низкоуровневые оптимизации.
Главные требования: математическая корректность и минимальная стоимость gas.

## Поле

Модуль поля:

```text
p =
0x01c4c62d92c41110229022eee2cdadb7f997505b8fafed5eb7e8f96c97d873
  7fdb925e8a0ed8d99d124d9a15af79db117e776f218059db80f0da5cb537e38
  685acce9767254a4638810719ac425f0e39d54522cdd119f5e9063de245e8001
```

Элемент поля хранится тремя 256-битными словами в little-endian порядке:

```text
x = x0 + 2^256 * x1 + 2^512 * x2
```

Слова модуля:

```solidity
uint256 constant P0 =
    0x685acce9767254a4638810719ac425f0e39d54522cdd119f5e9063de245e8001;
uint256 constant P1 =
    0x7fdb925e8a0ed8d99d124d9a15af79db117e776f218059db80f0da5cb537e38;
uint256 constant P2 =
    0x01c4c62d92c41110229022eee2cdadb7f997505b8fafed5eb7e8f96c97d873;
```

## Требуемая функция

Реализуйте библиотеку:

```solidity
library FieldMul3 {
    function mulMod3(
        uint256 a0,
        uint256 a1,
        uint256 a2,
        uint256 b0,
        uint256 b1,
        uint256 b2
    ) internal pure returns (
        uint256 r0,
        uint256 r1,
        uint256 r2
    );
}
```

Входы задают приведенные элементы:

```text
0 <= a < p
0 <= b < p
```

Результат должен задавать:

```text
r = a * b mod p
0 <= r < p
```

Можно использовать Solidity, inline Yul или их сочетание. Внешнее устройство
реализации не ограничивается.

## Тесты

Создайте минимальный Foundry-проект и автоматические тесты корректности.

Нужно проверить:

1. `0 * 0`.
2. `0 * x`.
3. `1 * x`.
4. `(p - 1) * (p - 1)`.
5. Значения с заполненными старшими словами.
6. Не менее `5 000` псевдослучайных пар элементов.
7. Для каждого результата условие `r < p`.

Эталонное значение вычисляйте независимо с помощью целочисленной арифметики
произвольной точности.

## Измерение gas

Добавьте benchmark, который выполняет не менее `2 048` последовательных
внутренних вызовов `FieldMul3.mulMod3` в цикле. Подготовку входов и вывод логов
не включайте в измерение.

Выведите:

```text
totalGas
gasPerOperation = totalGas / iterationCount
```

Целевая стоимость:

```text
не более 1 500 gas на одно внутреннее умножение
```
