# Задание: независимая реализация 3-limb умножения в Solidity/Yul

## 1. Цель

Нужно независимо, с нуля реализовать максимально экономное по gas умножение
элементов фиксированного 753-битного простого поля в Solidity/Yul.

Необходимо получить внутреннюю библиотечную функцию со стоимостью:

```text
не более 1 500 gas на одно умножение
```

Стоимость должна быть измерена для повторных внутренних вызовов функции в
цикле без учета развертывания контракта, подготовки входов и вывода логов.

## 2. Ограничение независимости

Не используйте и не анализируйте существующие реализации этой операции из
других проектов. Не переносите готовый код из внешних библиотек. Реализацию
нужно спроектировать самостоятельно.

В итоговом отчете не требуется обзор чужих решений. Требуется предоставить
собственный рабочий код, тесты корректности и измеренный результат.

## 3. Поле

Элементы поля задаются по модулю:

```text
p =
0x01c4c62d92c41110229022eee2cdadb7f997505b8fafed5eb7e8f96c97d873
  7fdb925e8a0ed8d99d124d9a15af79db117e776f218059db80f0da5cb537e38
  685acce9767254a4638810719ac425f0e39d54522cdd119f5e9063de245e8001
```

Каждый элемент хранится тремя 256-битными словами в little-endian порядке:

```text
x = x0 + 2^256 * x1 + 2^512 * x2
```

Слова модуля:

```solidity
uint256 constant P0 =
    0x685acce9767254a4638810719ac425f0e39d54522cdd119f5e9063de245e8001;
uint256 constant P1 =
    0x7fdb925e8a0ed8d99d124d9a15af79db117e776f218059db80f0da5cb537e38;
uint256 constant P2 =
    0x01c4c62d92c41110229022eee2cdadb7f997505b8fafed5eb7e8f96c97d873;
```

## 4. Математическая семантика

Для сравнимого эксперимента функция должна работать с Montgomery-представлением.

Пусть:

```text
B = 2^256
R = B^3
```

Входные слова кодируют приведенные значения:

```text
0 <= aMont < p
0 <= bMont < p
```

где:

```text
aMont = a * R mod p
bMont = b * R mod p
```

Функция должна вернуть приведенный результат:

```text
result = aMont * bMont * R^(-1) mod p
```

то есть Montgomery-представление произведения:

```text
result = a * b * R mod p
```

Для каждого корректного входа требуется:

```text
0 <= result < p
```

## 5. Обязательный API

Создайте библиотеку:

```solidity
library IndependentMontgomeryMul3
```

и реализуйте функцию:

```solidity
function montMul3(
    uint256 a0,
    uint256 a1,
    uint256 a2,
    uint256 b0,
    uint256 b1,
    uint256 b2
) internal pure returns (
    uint256 r0,
    uint256 r1,
    uint256 r2
);
```

Можно использовать Solidity, inline Yul или их сочетание. Внутреннее устройство
функции не ограничивается. Критерии выбора реализации: корректность и итоговая
стоимость gas.

Также создайте минимальный внешний harness:

```solidity
contract IndependentMontgomeryMul3Harness {
    function montMul3(
        uint256 a0,
        uint256 a1,
        uint256 a2,
        uint256 b0,
        uint256 b1,
        uint256 b2
    ) external pure returns (
        uint256 r0,
        uint256 r1,
        uint256 r2
    );
}
```

конфигурации.

## 7. Проверка корректности

Нужна независимая эталонная реализация на языке с целочисленной арифметикой
произвольной точности. Она должна вычислять:

```text
aMont * bMont * R^(-1) mod p
```

и сравнивать результат с Solidity/Yul-функцией.

Обязательные тесты:

1. `0 * 0`.
2. `0 * x`.
3. Montgomery-единица, умноженная на произвольное значение.
4. Значения `p - 1`, `p - 2`.
5. Значения с заполненными старшими словами.
6. Не менее `5 000` псевдослучайных пар приведенных элементов.
7. Проверка условия `result < p` для каждого теста.

Все тесты должны выполняться автоматически одной командой.

## 8. Измерение gas

Нужно измерить два режима.

### 8.1. Основной режим: внутренний вызов

Создайте benchmark-функцию, которая:

1. получает или задает два full-width приведенных Montgomery-элемента;
2. выполняет не менее `2 048` последовательных внутренних вызовов
   `IndependentMontgomeryMul3.montMul3`;
3. передает результат предыдущего умножения в следующее;
4. измеряет суммарный gas только внутри цикла;
5. выводит `totalGas / iterationCount`.

Именно это значение сравнивается с целевым пределом:

```text
<= 1 500 gas/op
```

### 8.2. Дополнительный режим: внешний вызов

Отдельно измерьте стоимость вызова через
`IndependentMontgomeryMul3Harness.montMul3`. Это число нужно привести в отчете,
но оно не используется как основной критерий.
