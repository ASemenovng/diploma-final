# Эксперимент с независимой реализацией 3-limb умножения

Директория содержит изолированную проверку реализации, полученной от Claude.
Контракт вычисляет канонический результат:

```text
r = a * b mod p
```

для 753-битного модуля MNT4-753. Элемент поля хранится тремя 256-битными
словами в little-endian порядке.

## Минимальное исправление исходного кода

В исходном Yul-коде локальная переменная `b1` внутри `redc` конфликтовала с
одноименным входным параметром Solidity-функции. Для компиляции она
переименована в `borrow1`. Алгоритм не изменен.

## Проверка корректности

Эталонная реализация находится в `scripts/ref_mul.py` и использует целые числа
Python произвольной точности. Контракт проверяется на граничных случаях и на
5000 детерминированно сгенерированных случайных векторах:

```bash
forge test --match-test 'test_(Zero|One|PMinus|High|Random64)' -vv
forge test --match-test test_Random5000AgainstPython -vv
```

## Измерение газа

Буквальный benchmark из ответа Claude:

```bash
forge test --match-contract BenchClaudeExact -vv
```

Отдельный benchmark на полноразмерных элементах поля:

```bash
forge test --match-contract BenchClaudeFullWidthExact -vv
```

На текущей конфигурации Foundry получены значения:

| Сценарий | Gas/op |
|---|---:|
| Буквальный benchmark Claude с малыми входами | 5 121 |
| Отдельный benchmark с полноразмерными входами | 5 643 |

Для оценки реализации следует использовать второе значение: оно не занижает
стоимость за счет малых входных слов.

## Важное ограничение сравнения

Эта функция выполняет каноническое умножение `a * b mod p`. Производственный
hot path MNT4 измеряет одну операцию Montgomery-domain умножения:

```text
aMont * bMont * R^-1 mod p
```

Поэтому значение этого эксперимента нельзя напрямую сравнивать с
`2 959 gas/op` для одной внутренней Montgomery-операции без учета преобразования
формата результата.
