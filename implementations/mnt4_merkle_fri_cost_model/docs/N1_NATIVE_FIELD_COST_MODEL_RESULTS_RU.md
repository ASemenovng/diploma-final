# N1: Rust cost-model нативной MNT4 Merkle/FRI-схемы

## Статус

Модель реализована до написания Solidity verifier-а. Она не подменяет реальный gas-report: нижняя оценка содержит обязательные данные и FRI-арифметику, ожидаемая оценка дополнительно использует явно перечисленные инженерные бюджеты.

DEEP-FRI строки ниже являются экспериментальными. Без отдельной инстанциации конкретной DEEP-FRI теоремы ни один такой профиль не является production-профилем.

## Строгий ordinary-FRI профиль

- `rho = 1/8`, `delta0 = 0.1507257`;
- минимум запросов для `128 bit`: `544`;
- выбранный production-профиль: `576` запросов, FRI-вклад `135.762 bit`;
- schedule: `[2, 2, 2, 4]`, последний слой `32`;
- раскрыто source rows: `1698`;
- раскрыто FRI field values: `6152`;
- Merkle frontier FRI: `2689` хешей;
- обязательная calldata: `2072224` байт;
- сегментов LDE-домена: `1`.

## Ограничения N1

- Это Rust cost-model, а не измерение Solidity.
- `relation_muls_per_query`, технический Keccak budget и control-flow budget являются явными допущениями.
- Normal basis пока не выбран: его преимущество должно подтверждаться индексатором и измерением конверсий.
- DEEP-FRI нельзя использовать для production stop/go до численной инстанциации его теоремы.
