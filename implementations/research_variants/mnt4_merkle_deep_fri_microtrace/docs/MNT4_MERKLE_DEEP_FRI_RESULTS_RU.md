# Результаты реализации Merkle/DEEP-FRI микротрассы для MNT4-753

## 1. Что реализовано

Модуль реализует неинтерактивную проверку уравнения сопряжений

\[
e(P,Q)\cdot e(-R,S)=1,
\]

где точки \(Q,S\in G_2\) фиксированы конфигурацией, а точки
\(P,R\in G_1\) передаются пользователем.

В отличие от `article640_mnt4`, Solidity-контракт не исполняет прямой цикл
Миллера. Rust backend строит пошаговую residue-рекурсию, а контракт проверяет
корректность раскрытых фрагментов трассы через AIR, Merkle-multiproof и
DEEP-FRI.

Реализованы:

1. точное расписание из `1500` реальных микроопераций;
2. дополнение до трассы длины `2048`: `547` HOLD-переходов и STOP-строка;
3. фиксированная таблица коэффициентов линий для \(Q,S\);
4. LDE на домене размера `32768`;
5. quotient для объединенного AIR-равенства;
6. неинтерактивный Fiat-Shamir transcript;
7. компактные Merkle-multiproof без повторной передачи frontier-хешей;
8. DEEP-значения и восемь FRI-folding раундов;
9. сериализация `proof.bin`;
10. Solidity-parser, который читает `bytes calldata` без копирования proof целиком;
11. положительные и отрицательные Foundry-тесты.

## 2. Rust backend

Команды:

```bash
cd rust/microtrace_backend
cargo test --release
cargo run --release --bin build_fixed_config -- artifacts/fixed-reproducible
cargo run --release --bin prove_fixture -- artifacts/benchmark-32q benchmark-32q
cargo run --release --bin prove_fixture -- artifacts/conservative-128q conservative-128q
```

Последние сохраненные метрики:

| Профиль | Запросы | Размер proof | Верхняя оценка calldata gas | Время генерации | Пиковая память |
|---|---:|---:|---:|---:|---:|
| `benchmark-32q` | 32 | 263 612 байт | 4 236 224 | 9 295 мс | 221,3 MiB |
| `conservative-128q` | 128 | 953 468 байт | 15 273 920 | 8 537 мс | 225,1 MiB |

Время зависит от текущей загрузки машины. Числа в таблице сохранены в
`metrics.json`; размер proof и структура открытий воспроизводятся
детерминированно.

## 3. Solidity-verifier

Основной метод:

```solidity
verifyEquationMicrotrace(P, R, c, cInv, proof) returns (bool)
```

Проверка выполняется в следующем порядке:

1. точки \(P,R\) принадлежат кривой;
2. свидетельства \(c,c^{-1}\) удовлетворяют \(c\cdot c^{-1}=1\);
3. заголовок proof имеет ожидаемые версию, профиль и размеры доменов;
4. transcript детерминированно восстанавливает challenges;
5. OOD-равенство связывает trace, fixed table и quotient;
6. Merkle-multiproof подтверждают раскрытия trace, fixed table, quotient и FRI;
7. раскрытые DEEP-значения вычисляются контрактом заново;
8. FRI-folding приводит значения к финальному многочлену степени меньше `8`;
9. контракт возвращает `true`.

## 4. Gas-сравнение

Для сравнения используется наиболее дешевый завершенный Article640 baseline:

```text
MNT4Article640FixedShardsVerifier.verifyEquationResidueFixedShards
```

Он хранит адреса data-shards в контракте и непосредственно исполняет
оптимизированный цикл Миллера с residue-проверкой.

### 4.1. Execution gas

| Реализация | Execution gas |
|---|---:|
| Article640 fixed-shards residue baseline с проверкой G1 | 93 734 789 |
| Merkle/DEEP-FRI `benchmark-32q` | 83 962 252 |
| Merkle/DEEP-FRI `conservative-128q` | 640 161 168 |

Профиль `benchmark-32q` снижает execution gas на

```text
93 734 789 - 83 962 252 = 9 772 537 gas
```

или примерно на `10,4%`.

### 4.2. Calldata

Для реальной транзакции необходимо учитывать calldata proof.

| Реализация | Execution gas | Верхняя оценка calldata gas | Сумма |
|---|---:|---:|---:|
| Article640 fixed-shards residue baseline с проверкой G1 | 93 734 789 | малая фиксированная часть | около 93,7 млн |
| Merkle/DEEP-FRI `benchmark-32q` | 83 962 252 | 4 236 224 | 88 198 476 |
| Merkle/DEEP-FRI `conservative-128q` | 640 161 168 | 15 273 920 | 655 435 088 |

После учета worst-case calldata профиль `benchmark-32q` сохраняет выигрыш около
`5,9%`. Консервативный профиль существенно дороже прямого Article640 verifier-а.

### 4.3. Foundry test-call gas

Foundry также показывает стоимость всего тестового вызова:

| Профиль | Test-call gas |
|---|---:|
| `benchmark-32q` | 102 238 330 |
| `conservative-128q` | 726 684 188 |

Эти числа включают тестовую подготовку аргументов в памяти и не должны
смешиваться с `execution gas` метода контракта.

## 5. Проверки корректности

Rust:

```text
9 passed; 0 failed
```

Foundry:

```text
21 passed; 0 failed
```

Проверяются:

1. положительные proof для обоих профилей;
2. подмена \(P\), \(R\), \(c\), \(c^{-1}\);
3. неканоническое кодирование элемента поля;
4. подмена `configDigest`;
5. подмена OOD bundle и коэффициента финального многочлена;
6. подмена trace, fixed, quotient и FRI leaf;
7. подмена trace/deep root;
8. подмена DEEP-инверсии;
9. удаление, добавление и перестановка frontier-хешей.

## 6. Ограничения результата

Получен завершенный экспериментальный pipeline, но не готовый mainnet-verifier.

1. Runtime-код `MNT4MerkleDeepFriVerifier` имеет размер `31 012` байт при
   используемых настройках оптимизатора. Это выше лимита EIP-170
   `24 576` байт. Даже при `optimizer_runs=1` размер остается `27 825` байт.
2. Даже наиболее дешевый профиль не укладывается в типичный лимит газа блока.
3. Профили `32` и `128` запросов нужны для измерений. Production-стойкость
   требует независимого криптографического аудита и явной soundness-модели.

## 7. Вывод

Микротрасса действительно заменяет прямое on-chain исполнение цикла Миллера
проверкой полиномиальных отношений. Однако для MNT4-753 стоимость проверки
Merkle/DEEP-FRI раскрытий остается высокой. Исследовательский профиль дает
умеренное снижение gas относительно Article640 fixed-shards baseline, но
консервативный профиль проигрывает ему в несколько раз. Это подтверждает
основной компромисс: уменьшение объема арифметики требует передачи и проверки
достаточного числа криптографически связанных раскрытий.

После независимого аудита результат следует трактовать именно как
`microtrace reference baseline`, а не как доказательство оптимальности
Merkle/FRI-подхода. Текущий Solidity verifier содержит устранимые расходы, а
более агрессивное блочное сжатие из первоначальной спецификации пока не
реализовано. Подробности:

```text
docs/MNT4_MERKLE_DEEP_FRI_INDEPENDENT_AUDIT_RU.md
```
