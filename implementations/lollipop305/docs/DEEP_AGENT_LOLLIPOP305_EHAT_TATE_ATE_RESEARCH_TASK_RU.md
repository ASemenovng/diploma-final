# Задание для deep agent: вывести Tate/ate-style verifier для `Ehat/Fq2` в lollipop-305

## 1. Контекст задачи

В дипломной работе исследуется, можно ли построить более дешевую основу для будущего folding на циклических pairing-friendly конструкциях. Базовая линия работы:

1. Полное on-chain вычисление сопряжения на MNT-кривых слишком дорого для EVM.
2. Подход ePrint 2024/640 (`On Proving Pairings`) предлагает оптимизации проверки pairing equation:
   - заранее подготовленные линии цикла Миллера;
   - fixed-`Q`/prepared-line режим;
   - замену полной финальной экспоненты проверкой witness-отношения;
   - полиномиальную компрессию цикла Миллера как более продвинутый вариант.
3. Статья ePrint 2024/1627 (`Cycles of supersingular elliptic curves for pairing-based proof systems`) предлагает lollipop-конструкции, где часть арифметики может выполняться над меньшими полями.

В проекте исследуется пример `lollipop-305-158` из ePrint 2024/1627. Цель исследования: понять, можно ли реализовать для lollipop-305 полный аналог MNT4/MNT6 article640-verifier, но дешевле за счет 2-limb арифметики.

Важно: у тебя нет доступа к коду проекта. Это задание содержит весь необходимый математический и инженерный контекст.

Если будешь выполнять вычислительные проверки, используй быстрые реализации, предпочтительно Rust. Не используй Python для больших полевых/групповых проверок: на этих параметрах Python-реализации могут работать слишком долго. Допустимы Rust, Sage/Magma для математической проверки, либо самостоятельная Rust-реализация полей на `BigUint`.

## 3. Кривые из lollipop-305

В статье ePrint 2024/1627 для `x ≡ 10 mod 12` получается случай `(u,v)=(2,2)`.

### 3.1. Первая cycle-кривая

```text
E_cycle / Fp2:
    y^2 = x^3 + (mu + 1)x,
    mu^2 + 1 = 0.
```

Поле:

```text
Fp2 = Fp[mu] / (mu^2 + 1)
```

Целевое поле для order-`q` pairing:

```text
Fp4*
```

Для нее уже найден корректный distorted Tate/Article640-style путь:

```text
psi_E(x,y) = (theta^2 * x^p, theta^3 * y^p),
theta^4 = mu.
```

Проверка строится в стиле:

```text
F = f_{q,Q}(P) * f_{q,Q}(-P)
F^((p^4 - 1)/q) = 1
```

или через residue witness:

```text
c^q = F,
c * c^{-1} = 1.
```

### 3.2. Вторая cycle-кривая, которая требует исследования

```text
Ehat_cycle / Fq2:
    y^2 = x^3 + (eta + 2),
    eta^2 + 2 = 0.
```

Поле:

```text
Fq2 = Fq[eta] / (eta^2 + 2)
```

Статья ePrint 2024/1627 утверждает, что target group для order-`p` Weil pairing на `Ehat` лежит в:

```text
Fq6*
```

В проекте была найдена башня:

```text
Fq6 = Fq2[w] / (w^3 - rho)
rho^2 = (2 + eta)/(2 - eta)
```

и distortion/Frobenius map:

```text
psi_Ehat(x,y) = (w^2 * x^q, rho * y^q)
```

Проверено вычислительно:

```text
rho^2 = (2 + eta)/(2 - eta)
w^3 = rho
w^6 = (2 + eta)/(2 - eta)
psi_Ehat(Q) lies on Ehat(Fq6)
[p]psi_Ehat(Q) = O
```

## 4. Текущая проблема

Сейчас для `Ehat/Fq2` был реализован fallback через Weil equation:

```text
e_p(P,Q) = f_{p,P}(Q) / f_{p,Q}(P)
```

и проверка:

```text
e_p(P,Q) * e_p(-P,Q) = 1
```

В раскрытом виде:

```text
f_{p,P}(Q) * f_{p,-P}(Q)
=
f_{p,Q}(P) * f_{p,Q}(-P)
```

Эта проверка математически корректна как Weil-pairing relation. Но она не является тем же подходом, что MNT4/MNT6 article640 verifier, и она очень дорогая в EVM, потому что требует четыре Miller-трассы в `Fq6`.

Текущие измерения для этого fallback:

```text
Ehat Weil equation:
    около 200.6M gas по function gas-report,
    около 247.1M gas в test-call.
```

Это делает полный lollipop-cycle непрактичным в прямом on-chain режиме.

## 5. Ключевой вопрос исследования

Нужно выяснить:

> Можно ли для `Ehat_cycle/Fq2` реализовать reduced Tate/ate-style pairing verifier, аналогичный `E_cycle`, MNT4 и MNT6, то есть без Weil-ratio формы `f_{p,P}(Q)/f_{p,Q}(P)`?

Иными словами, нужно понять, существует ли корректная форма:

```text
F = Miller_output(P,Q)
```

или для pairing equation:

```text
F = Miller_output(P1,Q1) * Miller_output(P2,Q2)
```

такая, что:

```text
F^h = 1,
h = (q^6 - 1)/p,
```

и такая, что ее можно заменить article640-style residue check:

```text
F = c^p
```

или более оптимальной формой из ePrint 2024/640.

Если это возможно, нужно вывести точные формулы, группы, loop parameter, denominator elimination и алгоритм проверки. Если невозможно или невыгодно, нужно строго объяснить почему.

## 6. Важное различие про знаменатели

Необходимо явно различать два вида знаменателей.

### 6.1. Вертикальные знаменатели в Miller loop

В полной форме Miller step содержит:

```text
f <- f^2 * l_{T,T}(P) / v_{2T}(P)
```

Для reduced Tate/ate pairing во многих случаях вертикальные знаменатели можно удалить, потому что после финальной экспоненты они исчезают или попадают в подполе, убиваемое финальной экспонентой.

Нужно проверить, применим ли этот аргумент к `Ehat/Fq2` и выбранным `G1/G2`.

### 6.2. Знаменатель в Weil pairing

В Weil pairing:

```text
e_p(P,Q) = f_{p,P}(Q) / f_{p,Q}(P)
```

этот знаменатель является частью определения Weil pairing, а не просто вертикальной линией внутри Miller loop. Его нельзя автоматически удалить тем же аргументом, что и вертикальные знаменатели в reduced Tate/ate pairing.

Задача исследования как раз в том, чтобы понять, можно ли уйти от Weil-ratio формы к reduced Tate/ate-style форме.

## 10. Критерий готовности исследования

Исследование считается завершенным, если после него можно сделать одно из двух.

### Вариант A: можно продолжать реализацию

Есть строгая спецификация, по которой можно заменить текущий `Ehat` Weil fallback на Tate/ate/residue verifier:

```text
Rust backend -> prepared lines -> Solidity Miller accumulator -> residue-check
```

и ожидается существенное снижение gas относительно текущих `~200M` function gas.

### Вариант B: нужно оставить отрицательный результат

Доказано, что для `Ehat/Fq2` в lollipop-305 нельзя получить аналогичный MNT4/MNT6 article640-style verifier без Weil-ratio или без другой дорогой проверки.

Тогда в дипломе нужно честно написать:

```text
lollipop-305 уменьшает стоимость базовой 2-limb арифметики, но для второй supersingular cycle-кривой Ehat/Fq2 текущая корректная pairing equation требует Weil-ratio формы. Это уничтожает ожидаемый выигрыш в прямой EVM-проверке.
```

## 11. Важное предупреждение

Не нужно предлагать “просто использовать Weil pairing”, потому что этот fallback уже реализован и измерен. Он корректен, но слишком дорог.

Нужно исследовать именно вопрос:

```text

с prepared lines и residue-check, аналогично MNT4/MNT6?
```

Если да -- вывести полную спецификацию.

Если нет -- доказать, почему нет.
