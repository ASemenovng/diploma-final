# Исследовательские варианты lollipop-305

В этой директории находятся варианты, нужные для воспроизводимого сравнения, но
не выбранные как основной fixed-cache путь.

`Lollipop305CommittedCacheVerifier.sol` принимает подготовленные линии в
`calldata` и перед тяжелой арифметикой сверяет доменно-разделенный commitment.
Подход корректно связывает вычисление с зарегистрированным кэшем, но пользователь
повторно передает крупные blob при каждом вызове.

Основной fixed-cache путь расположен в
`../src/Lollipop305FixedShardsVerifier.sol`. Он фиксирует адреса data-контрактов
в конструкторе и читает их runtime-код инструкцией `EXTCODECOPY`.
